Tomcat

简介

Tomcat是常用的Web服务器之一，由Apache组织提供的，提供对jsp、Servlet的支持。它是一种轻量级的javaweb容器（服务器）。
Tomcat里面有多个服务，每个服务里面有多个侦听器。
如下图，有三个Http请求，同时访问这个服务。Http侦听到这三个请求后，创建了两个线程，让这两个线程并发处理用户请求。这些线程内部都包括Http请求处理器的业务逻辑。待请求处理完成之后，这些线程将封装的请求对象和响应对象交给容器，然后侦听器会回收这些线程，供其他请求使用（Tomcat运用了线程池技术）。

工作过程

① Tomcat将http请求文本接收并解析，然后封装成HttpServletRequest类型的request对象，所有的HTTP头数据读可以通过request对象调用对应的方法查询到。

② Tomcat同时会要响应的信息封装为HttpServletResponse类型的response对象，通过设置response属性就可以控制要输出到浏览器的内容，然后将response交给tomcat，tomcat就会将其变成响应文本的格式发送给浏览器

启动和关闭Tomcat

启动：
方法一:bin目录下，双击startup.bat
方法二：cd tomcat/bin目录下，catalina run
停止：
bin目录下，双击shutdown.bat

Tomcat改端口

找到conf/server.xml中修改port 

<Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443" />

部署web工程到Tomcat中

将工程目录拷贝到webapps下，然后访问时ip:port/工程名/目录下/文件名

IJ配置tomcat

File->Build，Execution,Deployment->Application Servers->+->Tomcat Server

Servlet技术

Servlet简介

  1. Servlet是接口
  2. JavaWeb三大组件：Servlet程序、Filter过滤器、Listener监听器
  3. Servlet是运行在服务器上的一个java小程序，它可以接收客户端发送过来的请求，并响应数据给客户端。
  4. 使用 Servlet，您可以收集来自网页表单的用户输入，呈现来自数据库或者其他源的记录，还可以动态创建网页。

Servlet生命周期

  1. 执行Servlet构造器方法 （第一次访问时候调用）
  2. 执行init初始化 （第一次访问时候调用）
  3. 执行service方法 （每次访问都调用）
  4. 执行destroy销毁方法 （在web工程停止的时候调用）

Tomcat和Servlet的区别

Tomcat 是Web应用服务器，是一个Servlet/JSP容器.。

Tomcat 作为Servlet容器，负责处理客户请求,把请求传送给Servlet，并将Servlet的响应传送回给客户，而Servlet是一种运行在支持Java语言的服务器上的组件。

Servlet最常见的用途是扩展Java Web服务器功能，提供非常安全的，可移植的，易于使用的CGI替代品。
