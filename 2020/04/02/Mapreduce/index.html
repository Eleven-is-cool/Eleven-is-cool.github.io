MapReduce工作原理

①Map阶段

  1. 将输入的多个分片（Split）由Map任务以完全并行的方式处理。

  2. 每个分片由一个Map任务来处理

  3. 每个Map任务对输入分片的记录按照一定的规则解析成对

  4. 进行文本处理的时候，文本每一行就执行一次Map

  5. 由于频繁的磁盘I/O会降低效率，因此Map任务的输出<key,value>对会首先存储在Map任务所在节点的内存缓冲区中

  6. 在数据溢写到磁盘之前，会对数据进行分区(Partition)。

    • 分区的数量与设置的Reduce任务的数量相同（默认为1，可以在编写MapReduce程序时修改）

    • 数据分区的划分规则：取<key,value>中对key的hashCode值，然后对Reduce任务数量求余数，余数为分区编号，分区编号相一致的<key,value>则属于同一个分区

    • MapReduce提供默认的分区类:HashPartitioner

    • 分区后，会对同一分区中的<key,value>对按照key进行排序，默认升序

②Shuffle阶段

将Map任务的无规则输出形成一定的有规则数据，以便Reduce任务进行处理。<key,<1,1,1,1>>

③Reduce阶段

  1. 首先对Map阶段的输出结果按照分区进行再一次合并，将同一分区的<key,value>对合并到一起，然后按照key对对分区中的<key,value>对进行排序。

  2. 每个分区将排序后的<key,value>对按照key进行分组，key相同的<key,value>对合并为<key,value-list>对，最终每个分区形成多个<key,value-list>对。

  3. 排序并分组后的分区数据将会输入到reduce()方法进行处理，reduce()方法一次只能处理一个<key,value-list>对。

  4. 最后，reduce()将处理结果仍然以<key,value>对的形式通过context.write(key,value)进行输出。
