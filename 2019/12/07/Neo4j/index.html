Neo4j

发表于 2019-12-07 | 分类于 Neo4j |

Neo4j简介

Neo4j是一个高性能的,NOSQL图形数据库，它将结构化数据存储在网络上而不是表中。它是一个嵌入式的、基于磁盘的、具备完全的事务特性的Java持久化引擎，但是它将结构化数据存储在网络(从数学角度叫做图)上而不是表中。

Neo4j配置与启动

首先需要下载neo4j-community-3.5.13-unix.tar.gz
定位到文件所在位置，解压 tar -xf neo4j-community-3.5.13-unix.tar.gz
mv neo4j-community-3.5.13-unix /home/hadoop/neo4j
/home/hadoop/neo4j/bin/neo4j start
打开浏览器输入：http://localhost:7474

Neo4j基础操作

创建节点

CREATE (<node-name>:<label-name>{ <Property1-name>:<Property1-value>...<Propertyn-name>:<Propertyn-value> }),...
eg:CREATE (website:Demo{title:'Code',begin:2019-10-1,end:2019-12-1})

创建关系

CREATE (<node1-name>:<label1-name>)-[(<relationship-name>:<relationship-label-name>)]->(<node2-name>:<label2-name>)
eg:(Hong)-[:TASK_IN{roles:['boss']}]->(website)

获取有关节点与数据

MATCH (<node-name>:<label-name>)
单独使用无结果

返回结果

RETURN <node-name>.<Propertyn-name>
单独使用无结果

获取数据并返回结果

eg:MATCH (a:Person)-[r:Person_Relate{detial: ['classmate']}]->(b:Person) RETURN a,r,b
创建节点与关系必须同时在一个命令行执行，否则需要MAtch先定位

###实际例子
CREATE (website:Demo{title:’Code’,begin:2019-10-1,end:2019-12-1}),
(Keanu:Person{name:’Keanu Reeves’,born:1994}),
(Carrie:Person {name:’Carrie-Anne Moss’, born:1997}),
(Laurence:Person {name:’Laurence Fishburne’, born:1999}),
(Anndy:Person {name:’Andy Wachowski’, born:1995}),
(Hong:Person{name:’Lan Hong’, born:1980}),
(Hong)-[:TASK_IN{roles:[‘boss’]}]->(website),
(Keanu)-[:TASK_IN{roles:[‘Front-end’]}]->(website),
(Carrie)-[:TASK_IN{roles:[‘Back-end’]}]->(website),
(Laurence)-[:TASK_IN{roles:[‘Front-end’]}]->(website),
(Anndy)-[:TASK_IN{roles:[‘Back-end’]}]->(website),
(Hong)-[:Person_Relate{detial:[‘Teacher and Student’]}]->(Keanu),
(Hong)-[:Person_Relate{detial:[‘Teacher and Student’]}]->(Carrie),
(Hong)-[:Person_Relate{detial:[‘Teacher and Student’]}]->(Laurence),
(Hong)-[:Person_Relate{detial:[‘Teacher and Student’]}]->(Anndy),
(Carrie)-[:Person_Relate{detial:[‘classmate’]}]->(Anndy),
(Keanu)-[:Person_Relate{detial:[‘classmate’]}]->(Anndy)
match (a) return a查找所有关系
match (a:Person)-[r:Person_Relate{detial:[‘classmate’]}]->(b:Person) return a,r,b
查找关系为’classmate’的节点
match (a:Person)-[r:TASK_IN{roles:[‘Front-end’]}]->(b:Demo) return a,r,b
查找在项目中负责前端任务的人
